# Microfabrication Session Skill

## 目标

把现场自然语言记录整理成可复盘、可生成日总结/周总结/组会材料的微加工 session。

核心不是写漂亮总结，而是保留：

- 实际发生了什么。
- 用了什么样品、设备、recipe、装载方式和后处理。
- 在什么加工平台上完成，设备名称是什么，腔室、夹具、载片、压环、靶枪等关键部件与样品如何对应，并保存可用于复盘/汇报的设备或装载照片。
- 观察到什么异常。
- 现场如何判断和决策。
- AI 当场追问了什么、给了什么建议。
- 后续测量/继续加工应该关注什么。
- 哪些图片可以用于图文汇报。

## 默认目录结构

新建微加工 session 时，优先放在：

```text
workspace/experiments/YYYY-MM-DD/<short_process_name>/
  session.md
  images/
```

图片、显微图、加工后照片应保存到 `images/` 下，不只保留聊天附件、Lark 缓存路径或临时文件路径。

图片命名建议：

```text
001_formal_sample_loaded_quartz_ring.jpg
002_chip_16die_after_cleaving.jpg
003_small_hole_residue_after_etch.jpg
```

## 现场记录分类

整理记录时必须区分：

- 事实记录：已经发生的操作、测量结果、实际参数。
- 参数记录：设备、recipe、气压、功率、流量、温度、时间、cycle 数。
- 观察现象：颜色变化、刻透、油膜、粘连、裂纹、残留、透光、显微镜观察。
- 现场判断：用户或工程师对原因、风险、趋势的判断。
- AI 建议：AI 给出的解释、风险提醒、下一步建议。
- 待确认事项：影响计算、判断或后续决策，但当前还不知道的信息。

不能把观察、推测和结论混写成同一类。

## 即时追问规则

发现影响计算、归类、参数有效性或后续决策的信息缺口时，应当场追问，不要拖到日总结。

追问要少而关键，通常 1-3 个问题即可。优先问：

- 会影响速率/选择比计算的问题。
- 会影响样品归类的问题。
- 会影响异常因果判断的问题。
- 会影响下一步是继续加工还是转入测量的问题。

如果用户知道，就补入 session；如果用户不知道，就标记为：

- 待现场复查。
- 待询问工程师。
- 待回看设备日志。
- 暂不影响当前结论。

## 现场协作节奏

现场密集记录时，优先保证用户输入不断流，不要每条记录都立刻写入 `session.md`。

- 先在对话中轻量归类，必要时追问 1-2 个影响复现、安全、参数或流程时机的关键问题。
- 图片先标记为待保存证据，不必每张图都立即复制落盘。
- 等一段内容收口，或用户明确说“整理一下 / 写进 session”，再批量保存图片、更新 `session.md`。
- 正式整理时应按真实工艺流程重排，而不是按聊天顺序简单拼接。
- 收口时优先生成短 checklist，方便下次现场直接使用。

## 参数与样品绑定

参数变更必须绑定对象和轮次。

正确：

```markdown
- 纯 SiO2 测试片原计划尝试 9 Pa；因不好启辉，根据工程师经验改为 7 Pa；7 Pa 下启辉成功。
- 正式样品加工仍使用 9 Pa。
```

不够好：

```markdown
- 气压从 9 Pa 改为 7 Pa。
```

速率标定片必须明确它是否为正式样品。若只是材料消耗速率测试片，应写成“标定片/测试片”，不要和正式样品流程混写。

## 测量与计算

只要记录目标是算速率，至少需要：

- 刻前数值。
- 刻后数值。
- 有效刻蚀时间或 cycle 数。
- 测量方法。
- 测量点位置。

可以做简单计算，但要写清条件和边界。

不同条件下的数据可以做工程量级对比，但必须标注为“非严格对比”，并写明差异来源。

例如：

```markdown
严谨结论：
- SiO2 数据来自 7 Pa，Si 数据来自 9 Pa，不能作为严格选择比。

工程量级参考：
- Si 约 1200 nm/cycle，SiO2 约 3.07 nm/cycle，粗略量级约 391:1。
```

## 装载与异常

对碎片样品、异形样品或临时装载样品，必须记录：

- 样品是否碎裂、如何拼合。
- 固定/导热介质是什么。
- 固定介质具体放在哪里。
- 载片、铝托、压环、遮挡区域。
- 是否可能翘起、移动、局部接触不良。
- 装载后实际观察是否翘起或移动。

使用硅油、导热胶、碳胶、胶带等临时介质时，必须记录使用轮次、使用位置、清洁状态和复用历史。

若出现变黑、粘连、油膜、局部污染，应优先记录异常位置与固定介质接触区域是否空间相关。空间相关可以写成“高度怀疑相关”，除非有额外证据，不写成已证实因果。

## 刻透与百分比描述

“刻透”必须记录判据：

- 肉眼透光。
- 显微镜。
- 背面观察。
- 台阶仪。
- 截面或其它方式。

百分比描述必须写清分母：

- 单个开口面积的比例。
- 同类结构数量比例。
- 样品面积比例。
- 可测试 die 比例。

例如：

```markdown
- 对单个小洞开口而言，约 80% 面积已经刻透，边缘仍有残留。
```

## 图片证据

现场图片是 session 资产，不是附件。

每张图片至少记录：

- 保存路径。
- 对应阶段。
- 图片能证明什么。

Markdown 中应直接嵌图：

```markdown
![正式样品装载状态](images/001_formal_sample_loaded_quartz_ring.jpg)
```

周总结、组会材料优先从 session 的图片证据中选图，避免后续重新翻聊天记录。

## 生成 session.md 前的短收口

正式生成 `session.md` 前，应做一次短收口追问。问题不要贪多，通常问 2-4 个：

- 有没有关键照片、显微图或加工后照片。
- 最终 chip / die 数量和大致可测数量。
- 哪些异常区域后续还要测试，哪些应单独标注。
- 下一步更可能是测量、继续加工，还是重做样品。

然后给出 AI 的下一步判断和建议：

- 判断后续阶段：继续加工 / 转入测量 / 清洗后处理 / 数据整理。
- 说明建议依据。
- 给出少量可执行建议。

## session.md 推荐结构

```markdown
# YYYY-MM-DD <工艺主题> session

## 基本信息
## 样品层级与命名
## Recipe / 参数
## 标定片 / 测试片
## 正式样品装载
## 加工过程
## 异常观察与现场判断
## 后处理 / 清洗 / 裂片
## 图片证据
## 后续测试计划与关注点
## 本轮结论
## 待确认事项
## 现场记录方法沉淀
```

结构可以按实际任务删减，不要为了模板写空段落。
